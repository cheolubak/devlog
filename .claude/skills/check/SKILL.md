# 빌드 및 린트 검증

프로젝트의 코드 품질과 빌드 상태를 한 번에 검증한다.

## 실행 순서

다음 명령을 순차적으로 실행한다. 각 단계에서 에러가 발생하면 해당 에러를 보고하고 수정을 시도한다.

### 1단계: 린트 검사

```bash
pnpm lint
```

- ESLint로 코드 스타일 및 규칙 검사
- 에러가 있으면 자동 수정 가능한 것은 수정하고, 수동 수정이 필요한 것은 보고

### 2단계: 테스트 실행

```bash
pnpm test
```

- Vitest 단위 테스트 실행
- 실패한 테스트가 있으면 원인 분석 및 보고

### 3단계: 프로덕션 빌드

```bash
pnpm build
```

- Next.js 프로덕션 빌드 (TypeScript 타입 체크 포함)
- 빌드 에러가 있으면 원인 분석 및 보고

## 결과 보고 형식

```markdown
## 검증 결과

### 린트 (ESLint)
- ✅ 통과 / ❌ N개 에러
  - 에러 내용 (있는 경우)

### 테스트 (Vitest)
- ✅ N개 통과 / ❌ N개 실패
  - 실패 내용 (있는 경우)

### 빌드 (Next.js)
- ✅ 성공 / ❌ 실패
  - 에러 내용 (있는 경우)
```

## 에러 발생 시

1. 에러 내용을 분석한다
2. 자동 수정이 가능한 경우 수정을 시도한다
3. 수정 후 해당 단계를 다시 실행하여 확인한다
4. 수정이 어려운 경우 에러 내용과 수정 방향을 보고한다
